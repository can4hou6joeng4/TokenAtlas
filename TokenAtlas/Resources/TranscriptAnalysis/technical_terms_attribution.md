# Built-in Technical Terms: Notes and Attribution

The bundled `technical_terms.json` file is a curated keyword list for transcript analysis. It contains only term labels, aliases, weights, categories, and tags. It does not copy definitions, descriptions, examples, cheatsheet bodies, or command explanations from external sources.

The list was informed by common terminology found in public documentation and open source reference projects, including:

- U.S. Web Design System (USWDS): https://github.com/uswds/uswds
  - License notes: primarily U.S. public domain / CC0, with third-party exceptions documented in the USWDS license file.
- GOV.UK Design System: https://design-system.service.gov.uk/
  - License notes: Open Government Licence v3.0, except where otherwise stated.
- Material Design / Material Web: https://github.com/material-components/material-web
  - License notes: Apache License 2.0 for Material Web components.
- GitLab Pajamas Design System: https://design.gitlab.com/
  - License notes: MIT License for the public Pajamas repository.
- IBM Carbon Design System: https://github.com/carbon-design-system/carbon
  - License notes: Apache License 2.0.
- iSAQB Glossary of Software Architecture Terminology: https://public.isaqb.org/glossary/glossary-en.html
  - License notes: Creative Commons Attribution 4.0 International.
- CNCF Cloud Native Glossary: https://github.com/cncf/glossary
  - License notes: Apache License 2.0 for code contributions; see repository license and project notes for content details.
- GitHub Docs: https://github.com/github/docs
  - License notes: CC-BY-4.0 for docs content and MIT for code.
- cheat/cheatsheets: https://github.com/cheat/cheatsheets
  - License notes: CC0 1.0.
- withfig/autocomplete: https://github.com/withfig/autocomplete
  - License notes: MIT License.
- tldr-pages: https://github.com/tldr-pages/tldr
  - License notes: CC-BY-4.0 for pages and MIT for scripts.

The bundled dictionary is intentionally a small metadata artifact derived from general technical vocabulary and local product needs, not a redistributed glossary or cheatsheet corpus.
